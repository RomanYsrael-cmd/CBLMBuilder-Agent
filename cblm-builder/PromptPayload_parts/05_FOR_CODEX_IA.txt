Pre-check:

If ./inbox is empty: EXIT immediately with no output.
Only accept CBLM Unit files in DOCX format named like:
  CBLM_Unit_<n>_<COURSE_CODE>_<Title>.docx
Example:
  CBLM_Unit_1_ICC_5_Information_Management.docx

Process ONE COURSE-CODE GROUP per run (grouped by code in the filename):
- If the inbox contains multiple course codes, deterministically pick ONE course-code group (sort by COURSE_CODE ascending, pick the first)
- For the chosen course code, process ALL unit files in ascending unit order

Goal:

Generate ONE IA for ONE course-code group (one syllabus/course):
- Build one IA payload JSON for the full course that lists all contents across all unit CBLMs
- Generate MIDTERM and FINALS exams (50 items each) and include them in the IA package
- Assemble ONE IA DOCX (IA_FULL.docx) using the IA templates in this order:
  00_FrontPage + 01_iaplan + 02_iaspesificinstruction + 03_midterm + 04_finals
- Do NOT generate CBLM DOCX files in this workflow
- Stop after one course-code group is completed (wait for user continuation)

General rules:

Do not ask unnecessary questions.
Prefer .\.venv\Scripts\python.exe for Python commands.
Use up to 600000 ms timeout for long-running terminal commands.
Follow the workspace files as the source of truth.
Do not edit the IA DOCX templates unless explicitly requested.
Important scope rule:
- This IA workflow starts from CBLM Unit DOCX inputs, not from the raw syllabus.
- Therefore it can assemble the IA package and course-level exams, but it cannot produce syllabus-derived TOS workbooks by itself.
- If the user specifically requires TOS-aligned exams driven by syllabus Topics, weeks, and `k/c/a` numbering, use the dedicated syllabus workflow separately:
  .\.venv\Scripts\python.exe .\tools\build_tos_and_exam_from_syllabus.py

Exam generation rule:
- Generate MIDTERM and FINALS locally during IA processing (no external API calls; do not require OPENAI_API_KEY).
- These IA exams are course-level packaged exams.
- If syllabus-derived TOS outputs already exist, report them, but do not overwrite them unless explicitly instructed.
Enforce IA uniqueness guardrails:
- IA project name and oral Q&A must be content-anchored (derived from the unit’s LO/content titles)
- Avoid repeating the same oral questions across units in the same course-code group (the final generic “performance criteria” question may repeat)
- If guardrails fail after retries, STOP with an error (do not move inputs)

Input:

One or more CBLM Unit DOCX files in ./inbox (potentially multiple course codes).

Extraction rules (from the CBLM Unit DOCX):

- qualification_title: extract from the CBLM front matter "Qualification Title" field (prefer table-based extraction). Do NOT use generic headers like "COMPETENCY-BASED LEARNING MATERIALS" as the qualification.
- current_unit.unit_of_competency: extract from the "The unit of competency, “...”" sentence or the MODULE CONTENT section.
- current_unit.module_title: extract from the module title line in front matter / MODULE CONTENT.
- current_unit.next_unit_of_competency: extract from the "You need to complete this module before..." sentence.
- current_unit.learning_outcomes[]:
  - Each LO section title in the CBLM becomes one learning_outcomes item:
    - index: 1..n (in order of appearance)
    - title: LO title text (no rewriting)
    - contents[]:
      - Each content/topic under that LO becomes one content item:
        - index: 1..m (in order)
        - title: content title as written in CBLM (no rewriting)

IA payload rules:

Create a NEW payload JSON that is compatible with tools/assemble_ia.py and the updated IA templates:

current_unit required fields:

  - interview_questions (array of exactly 5 strings)

Important wording rule (for IA template {{Contents}} ONLY):

When populating IA plan and rating sheet rows, the wording must fit:
"The evidence must show that the trainee can {{Contents}}."

Therefore, generate a derived "ability phrasing" for each content title when filling {{Contents}}:
- If the content title starts with a verb (e.g., Apply/Identify/Explain/Develop/etc.), keep the phrase but lowercase the first letter (e.g., "Apply X" -> "apply X").
- Otherwise use: "demonstrate competence in <content title>".

DO NOT modify the IA template placeholder {{Contents_Z}} meaning:
- {{Contents_Z}} remains the raw list of contents under the current LO (no wording adjustment).
