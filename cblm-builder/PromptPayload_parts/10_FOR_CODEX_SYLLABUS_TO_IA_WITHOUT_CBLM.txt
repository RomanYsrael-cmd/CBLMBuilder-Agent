CODEX must write the oral questions and acceptable answers directly into the IA payload JSON.

Create one IA payload for MIDTERM scope and one IA payload for FINALS scope.

The scope for MIDTERM and FINALS must be derived from the syllabus itself.
Do not use a naive half-split unless the syllabus structure explicitly supports it.
Use the same syllabus-derived scope boundaries for:
- the MIDTERM IA
- the FINALS IA
- the MIDTERM exam
- the FINALS exam
- the MIDTERM oral questioning
- the FINALS oral questioning

If the syllabus provides weeks, grading periods, prelim/midterm/finals labels, sequence boundaries, or other scope indicators, use them as the source of truth for term coverage.
If multiple syllabus signals exist, resolve the scope deterministically and keep IA, TOS, exams, and oral questioning aligned to the same term boundaries.

Each term-specific IA payload must:
- list only the contents that belong to that term's syllabus-derived scope
- preserve topic/content order within that term
- use content-anchored IA wording
- include the term value used by the front page template:
  - MIDTERM IA -> `term = MIDTERM`

- generate model oral questions and model answers grounded in the actual syllabus-derived topics, learning outcomes, content items, module title, and next-unit context when available

Important wording rule (for IA template {{Contents}} ONLY):

When populating IA plan and rating sheet rows, the wording must fit:
"The evidence must show that the trainee can {{Contents}}."

Therefore, generate a derived "ability phrasing" for each content title when filling {{Contents}}:
- If the content title starts with a verb, keep the phrase but lowercase the first letter.
- Otherwise use: "demonstrate competence in <content title>".

DO NOT modify the IA template placeholder {{Contents_Z}} meaning:
- {{Contents_Z}} remains the raw list of contents under the current LO.

IA template interpretation rule:

- `{{term}}` on the front page template must be populated with the current term:
  - `MIDTERM`
  - `FINALS`
- `{{Y}}. {{LO}}` means:
  - `{{Y}}` = Learning Outcome number within the current term-specific IA payload
  - `{{LO}}` = the current Learning Outcome title
- `{{Y}}.{{Z}}. {{Contents}}` means:
  - `{{Y}}` = Learning Outcome number
  - `{{Z}}` = topic number under that Learning Outcome
  - `{{Contents}}` = the current topic title, converted to the required ability phrasing only when needed by the destination template sentence
- `{{Contents_Z}}` means:
  - the full raw list of all subtopics under the current topic/Learning Outcome entry

IA generation rule:

- Build one MIDTERM IA payload JSON and one FINALS IA payload JSON directly from the syllabus structure.
- Generate 5 model oral questions for MIDTERM and 5 model oral questions for FINALS.
- For each oral question, write one model acceptable answer that directly answers the question using the real extracted syllabus content.
- The acceptable answer must be concrete and specific.
- Example:
  - Question: `What is 1 + 1?`
  - Acceptable answer: `2`
- If the question asks for a title, the acceptable answer must be the exact title.
- If the question asks for a content item, the acceptable answer must be the exact content item.
- If the question asks for the next unit, the acceptable answer must be the exact next unit when present.
- Avoid generic wording like "Correctly explains..." or "Describes a logical sequence..." in acceptable answers.
- Save the MIDTERM IA payload under `state/ia_payloads/<COURSE_CODE>/IA_MIDTERM.json`.
- Save the FINALS IA payload under `state/ia_payloads/<COURSE_CODE>/IA_FINALS.json`.
- Assemble `output/ia/<COURSE_CODE>/IA_MIDTERM.docx` using `tools/render_term_ia_from_authored_state.py`.
- Assemble `output/ia/<COURSE_CODE>/IA_FINALS.docx` using `tools/render_term_ia_from_authored_state.py`.
- Do not require CBLM unit DOCX files as inputs for this workflow.

TOS-aligned exam rule:

After the syllabus structure is extracted, run the dedicated syllabus-to-TOS planning workflow:

.\.venv\Scripts\python.exe .\tools\build_tos_plan_from_syllabus.py

This planning workflow must:
- identify all syllabus Topics
- resolve which Topics belong to MIDTERM and FINALS
- read week counts when present and map them to `No. of Days Taught`
- use those week counts as the basis for TOS topic weighting
- allocate `No. of Items` per Topic proportionally
- force each term total to exactly 50:
  - if the total is greater than 50, subtract 1 from one or more Topics until the total is 50
  - if the total is less than 50, add 1 to one or more Topics until the total is 50
- create one exam objective per Topic with 10-20 words
- fill the term-specific TOS templates:
  - `templates/TOS_TEMPLATE_MIDTERMS.xlsx`
  - `templates/TOS_TEMPLATE_FINALS.xlsx`
  and populate their placeholders for:
  - topic
  - objective
  - number_of_days
  - k_array
  - c_array
  - a_array
- write the term state JSON files that CODEX will complete with authored MCQs

Exam authoring rule:

- CODEX must write the 50 MIDTERM MCQs directly into `state/tos/<COURSE_CODE>/MIDTERM.json`
- CODEX must write the 50 FINALS MCQs directly into `state/tos/<COURSE_CODE>/FINALS.json`
- the tools must not invent the exam wording for this workflow
- each authored MCQ must include:
  - `question`
  - `a`
  - `b`
  - `c`
  - `d`
  - `answer`
  - `level`
- `level` must be one of:
  - `knowledge`
  - `comprehension`
  - `application`
- CODEX must preserve the authored MCQ order to match the TOS numbering arrays exactly:
  - question 1 must match the level expected by the TOS arrays
  - question 2 must match the level expected by the TOS arrays
  - and so on through question 50
- do not let comprehension collapse to zero when the TOS plan expects comprehension items
- after CODEX authors the MCQs, render the exam DOCX files with:

- the rendered exam DOCX must append a model answer key after the 50 multiple-choice items
